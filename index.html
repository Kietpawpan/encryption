Advanced Encryption Standard

โปรแกรม ENCRYPT 72 ใช้สำหรับเข้ารหัสข้อมูล ด้วยวิธี Advanced Encryption Standard (AES) เพื่อคุ้มครองข้อมูลข่าวสารที่มีชั้นความลับ ระดับลับ (CONFIDENTIAL) ลับมาก (SECRET) และลับที่สุด (TOP SECRET) รายละเอียดตามนโยบายและข้อเสนอแนะของคณะกรรมการด้านระบบความปลอดภัยแห่งชาติ (Committee on National Security Systems: CNSS) และสำนักงานความมั่นคงปลอดภัยไซเบอร์และโครงสร้างพื้นฐานของประเทศสหรัฐอเมริกา (Cybersecurity and Infrastructure Security Agency) ทั้งนี้ AES ได้เข้ามาตรฐาน ISO/IEC 18033-3

เทคนิคการเข้ารหัส
  1. private key ที่ใช้ล็อคและปลดล็อคข้อมูล มีความยาวเป็นบิท (bit) โดย 1 ตัวอักษร เท่ากับ 8 บิท หรือ 1 ไบต์ (byte) เช่น “a” คือ 01100001 ทั้งนี้ ความยาวของ private key จะเป็นตัวกำหนดความยาวของ round keys (กุญแจที่ใช้แปลงข้อมูลในแต่ละรอบ) ให้มีขนาด 128 บิท 192 บิท หรือ 256 บิท แล้วแต่กรณี และการเข้ารหัสจะใช้ข้อมูลต้นฉบับ (plaintext) ผสมกับ round key รอบละ 1 ดอก โดยมีการตัดข้อมูลเป็น block แต่ละ block ถูกตัดย่อยเป็นชิ้น ๆ แล้วสลับที่กัน ไม่ให้แปรผันแบบเส้นตรงกับข้อมูลเดิม (nonlinear complex modification) และมีการเปลี่ยน round key ทุกรอบ โดยที่จำนวนรอบการเข้ารหัส คือ 10 12 และ 14 รอบ สำหรับ round keys ขนาด 128 192 และ 256 บิท ตามลำดับ
  2. โปรแกรมนี้ใช้ crypto-js มาตรฐาน ในการเข้ารหัสแบบ AES โดยใช้ JavaScript Library จาก Github repository (crypto-js เวอร์ชัน 4.2.0)
  3. ในปี พ.ศ. 2567 สำนักงานความมั่นคงปลอดภัยไซเบอร์และโครงสร้างพื้นฐานของสหรัฐอเมริกา (Cybersecurity and Infrastructure Security Agency) ยังคงแนะนำให้หน่วยงานรัฐบาลสหรัฐอเมริกา ใช้วิธี AES ในการเข้ารหัสข้อมูลข่าวสารลับ แทนมาตรฐานเดิม (Data Encryption Standard หรือ DES)
  4. คณะกรรมการด้านระบบความปลอดภัยแห่งชาติของประเทศสหรัฐอเมริกา (U.S. Committee on National Security Systems: CNSS) มีนโยบายให้ใช้ AES เข้ารหัสข้อมูลข่าวสารลับ ระดับลับที่สุด (TOP SECRET) โดยใช้ key ขนาด 192 หรือ 256 บิท ส่วน key ขนาด 128 บิท ใช้เข้ารหัสข้อมูลข่าวสารลับ ระดับลับมาก (SECRET)
  5. โปรแกรมนี้ เขียนด้วยภาษา HTML/JavaScript แบบ open source ไม่มีการบันทึกข้อมูลที่เข้ารหัสไว้แต่อย่างใด หากท่านลืม private key และได้ทำลายข้อมูลต้นฉบับแล้ว จะไม่สามารถกู้ข้อมูลต้นฉบับได้ เว้นแต่จะคาดเดา private key ได้อย่างถูกต้อง
  6. ควรกำหนด private key ให้คาดเดาได้ยาก และเก็บเป็นความลับ ให้เข้าถึงได้เฉพาะผู้มีหน้าที่เกี่ยวข้องเท่านั้น
  7. สำหรับ Developer หากจำเป็นต้องใช้ private key ใน source code ควรนำ private key นั้น มาทำการ hash แล้วบันทึก hashed text แทน โดยใช้ Secure Hash Algorithm (SHA) ที่ปลอดภัยสูง เช่น SHA3-512
